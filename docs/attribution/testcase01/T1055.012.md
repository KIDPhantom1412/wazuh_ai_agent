# 攻击溯源智能体测试样例01
## 本测试样例基于redcanaryco/atomic-red-team项目中的[T1055.012](https://github.com/redcanaryco/atomic-red-team/blob/master/atomics/T1055.012/T1055.012.md)的Atomic Test #1: Process Hollowing using PowerShell检测攻击溯源智能体的效果

### 一、攻击介绍
1. 攻击环境：Windows
2. 攻击工具：本次攻击主要依赖于由 FuzzySecurity 编写的开源 PowerShell 脚本 [Start-Hollow.ps1](https://raw.githubusercontent.com/FuzzySecurity/PowerShell-Suite/720d8fe82396faf74f2ca19a3fe99a5c262a14b9/Start-Hollow.ps1)
3. 攻击描述：进程镂空 (Process Hollowing) 是一种旨在规避基于进程的安全防御的高级注入技术。其核心原理是：攻击者首先利用 Windows 原生 API（如 CreateProcess）启动一个合法的系统或应用程序进程，并将其主线程设置为挂起状态。接着，攻击者调用 ZwUnmapViewOfSection 或 NtUnmapViewOfSection 等 API，将该合法进程原本加载在内存中的代码区块清空（即“镂空”）。随后，攻击者使用 VirtualAllocEx 在该进程中重新分配内存，利用 WriteProcessMemory 将恶意的载荷代码写入其中。最后，通过 SetThreadContext 调整线程的执行上下文以指向恶意代码的入口点，并调用 ResumeThread 唤醒线程。


### 二、攻击过程描述
1. 脚本加载与伪造目标定位   
攻击者首先在 PowerShell 会话中加载了进程镂空脚本。为了掩蔽后续生成的恶意进程，攻击者计划实施父进程欺骗（Parent PID Spoofing）。为此，攻击者执行命令，在系统中查找当前正在运行的、合法的 explorer.exe（资源管理器）进程，并提取其进程标识符（PID）保存至变量中。
```
. "C:\AtomicRedTeam\atomics\T1055.012\src\Start-Hollow.ps1"
$ppid=Get-Process explorer | select -expand id
```
 2. 原生机制滥用与动态编译

在发起注入前，Start-Hollow.ps1 脚本需要调用底层的 Windows API（如 NtCreateProcessEx、WriteProcessMemory 等）。PowerShell 利用原生的 Add-Type 功能，在后台自动调用系统自带的 C# 编译器（csc.exe）动态编译所需的底层接口代码。在此过程中，powershell.exe 派生出 csc.exe 子进程，并在用户的临时目录（AppData\Local\Temp）下生成用于编译的源代码文件（.cs）和输出文件（.dll）。编译完成后，临时文件通常会被清理。这一原生机制被攻击者滥用于构建底层的注入调用环境。
 3. 触发攻击与父进程欺骗 
   
环境构建完成后，攻击者调用 Start-Hollow 函数，并在命令行参数中明确指定攻击的核心要素：被镂空的宿主程序（notepad.exe）、实际执行的恶意载荷（cmd.exe）以及用于伪装的父进程（explorer.exe）。

```
Start-Hollow -Sponsor "C:\Windows\System32\notepad.exe" -Hollow "C:\Windows\System32\cmd.exe" -ParentPID $ppid -Verbose
```
下图展示了借助 Invoke-AtomicRedTeam 模块实现的对 T1055.012（进程镂空）技术的标准化威胁模拟。 Invoke-AtomicTest 命令作为一个自动化测试封装器，将复杂的底层 Native API 调用、C# 动态编译过程以及目标注入参数高度集成化，通过执行这条简短的指令，即可在受害主机上模拟触发完整的进程镂空攻击链。
![攻击效果展示](images\testcase01_p1攻击脚本模拟结果.png)
 4. 宿主进程生成与内存注入

接收到执行命令后，脚本通过此前编译的底层 API 实施进程镂空操作：脚本以后台静默方式启动目标宿主程序 notepad.exe，并将其主线程置于挂起状态。同时，该 notepad.exe 的父进程属性被篡改，指向之前获取的 explorer.exe。随后，powershell.exe 跨进程访问处于挂起状态的 notepad.exe 的内存空间，清空其原有的合法代码，并将 cmd.exe 的执行代码及参数写入该空间。最终完成内存替换和进程环境块的重写后，脚本唤醒 notepad.exe 的主线程。至此，攻击执行完毕。系统中生成了一个名为 notepad.exe、父进程指向 explorer.exe 的进程，但其内存空间已被替换，实际运行的是攻击者指定的 cmd.exe 载荷。

2. 攻击溯源自动化调查 

将可疑日志的一些关键信息交给攻击溯源智能体即可进行调查，例如:"从agent 005发现一个可疑进程，进程pid为9272，帮我对其进行攻击溯源。请将查询的时间范围限定在 2026年3月30日的 9:30 到 9:40 之间。"界定调查的时间区间可以缩小日志搜索范围，提升查询速度。

![攻击溯源调查过程](images\testcase01_p4_攻击溯源过程图.png)

攻击溯源智能体接收到可疑日志信息后，首先获取该日志对应进程（pid:9272）的创建日志，并在这一步获取到重要命令行信息：发现这是通过 Start-Hollow.ps1 脚本发起的进程镂空攻击，并直接从命令行中提取出了伪装宿主（notepad.exe）、注入载荷（cmd.exe）以及用于父进程欺骗的指令（-ParentPID）。

接着，智能体纵向查询，成功捕获到 PID 9272 派生了 csc.exe (PID 1028)，并紧接着查明了这两个进程在临时目录（Temp）下释放了 `.cs` 源码文件并将其编译为 `.dll` 恶意载荷的完整文件操作轨迹，准确甄别出这是利用 C# 编译器进行原生动态编译的防御规避行为。

然后，智能体开始横向深挖跨进程访问行为。通过查询进程注入与内存访问的相关日志，智能体发现恶意进程请求访问了 explorer.exe，并成功识别出对 `explorer.exe` 的高权限访问实为获取 PID 以实施“父进程欺骗”的前置动作
